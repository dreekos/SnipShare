# Contributing to SnipShare

Thank you for your interest in contributing to SnipShare! We welcome contributions from the community.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/SnipShare.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `npm install`
5. Make your changes
6. Test your changes: `npm start`
7. Commit your changes: `git commit -m "Add your feature"`
8. Push to your fork: `git push origin feature/your-feature-name`
9. Open a Pull Request

## Development Setup

```bash
# Clone the repository
git clone https://github.com/dreekos/SnipShare.git
cd SnipShare

# Install dependencies
npm install

# Copy environment example
cp .env.example .env

# Start development server
npm run dev
```

## Code Style

- Use consistent indentation (2 spaces)
- Follow existing code patterns
- Add comments for complex logic
- Keep functions small and focused

## Pull Request Guidelines

- Describe what your PR does
- Reference any related issues
- Make sure all existing features still work
- Keep PRs focused on a single change

## Reporting Issues

When reporting issues, please include:

- Description of the issue
- Steps to reproduce
- Expected behavior
- Actual behavior
- Browser/Node.js version

## Questions?

Feel free to open an issue for any questions or discussions.
